


### [번들링]
```
사용자에게 전달할 파일을 묶어주는 과정
```
  

### [웹팩]
```
HTML, CSS, JS, IMAGE 등의 자원을 모두 각각의 모듈로 보고 이를 하나의 묶음으로 조합 해 번들링하는 도구
웹팩 외에도 parcel, rollup등 다른 번들러들도 있다
웹팩하는 이유 : 로딩 속도를 개선하기 위해서 브라우저에서 서버로 요청하는 파일의 숫자를 줄이기 위해서.
```




### [CSR vs SSR]
```
CSR (client side rendering)
 브라우저에서 js를 이용해서 웹페이지를 생성하고 업데이트를 하는 방식입니다.
반면 SSR은 server side rendering로, 서버에서 웹페이지를 생성해 브라우저로 전달하는 방식 입니다. 
CSR은 페이지에 변경점이 있을 때 서버까지 갔다와 전체 페이지를 다시 생성하지 않고 변경점만 업데이는 되는 면에서 장점을 가지고 있습니다.

SSR (server side rendering)
 검색엔진에 유리하다는 장점을 가지고 있습니다.  
검색엔진은 크롤링은 통해 정보를얻어오는데 이 웹 크롤러는 사이트들의 방문을 통해 이루어지는데 CSR의 경우 초기 빈 페이지로 로드 되기 때문에 
SEO 면에서 불리한 점을 가지고 있는 점 등에서 SSR방식이 유리한 점을 가지고 있습니다. (또, 크롤러는 페이지 로딩 속도도 고려 등) 
```
